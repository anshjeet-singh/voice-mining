# ACQ AI Onboarding Prompt Playbook (all 14 prompts, mined cover to cover)

Source: Acquisition.com's internal AI onboarding prompts. These are the exact
templates Hormozi's team uses to turn RAW CLIENT MATERIAL (call transcripts,
videos, lead magnets, offers) into finished assets. The meta-lesson is our
pipeline's core doctrine: mine real artifacts -> extract the patterns ->
generate NEW copy in the same voice. Never write from a blank page when a
real transcript, video, or winning asset exists.

## The transcript-mining prompts (#1-#6)

1. **Mother Script builder**: feed 10 WINNING call transcripts, extract the
   common spine into ONE master script: Opener/Rapport, Frame/Agenda,
   Discovery, Transition to Pitch, Offer + Value Stack, Top 3-5 Objections,
   Final Close. Winning patterns beat invented ones.
2. **Newsletter style analyzer**: extract a writer's Voice Profile, Writing
   Tics, Narrative Structure, Formatting, Persuasion Mechanics, CTA Style
   into a reusable template; generate new issues IN that voice.
3. **Failed-call objection analyzer**: from lost calls, extract primary and
   secondary objections, root causes, and 3-5 exact handling lines each;
   consolidate into a Top 10 objection guide. Losses are data.
4. **Universal writing style analyzer**: snapshot, voice/tone, structure,
   sentence-level, diction, rhetorical devices, formatting -> a style rules
   checklist any writer (or AI) can follow.
5. **Sales call -> VSL generator**: winning call transcripts become the VSL:
   Hook, Big Promise, Relatability, Pain Expansion, Hidden Truth/Big Reveal,
   New Mechanism, What Makes This Different, Future Pacing, [PROOF INSERT:
   type + what it proves] placeholders, Offer Breakdown, Value Stack,
   Objection Handling, Guarantee, CTA, Close.
6. **Sales manager feedback**: Stop/Start/Keep coaching per call, with exact
   replacement phrasing, red-flag deal killers, drills, and an "If I ran the
   call" rewrite. Rules: no generic advice, tie everything to the transcript,
   explain the sales psychology WHY.

## The style-extraction prompts (#7-#9)

7. **YouTube script style extractor**: from 1-5 liked video transcripts,
   extract Structure & Flow (hook style, act transitions, curiosity
   maintenance, climax, CTA), Tone & Voice, Narrative Devices (open loops,
   reframes, data moments, future pacing), Sentence & Delivery Style,
   Formatting Patterns -> reusable template (HOOK / INTRO FRAME / SETUP
   STORY / PROBLEM EXPANSION / REVEAL / CORE CONTENT / EMOTIONAL BEAT /
   PROOF / CLOSING LESSON / CTA) -> new scripts that feel like the same
   creator with completely new content. Derive the style, never the words.
8. **Shorts script extractor**: same method compressed for 15-60s vertical:
   Hook (0-2s: shock line, pattern interrupt, contrarian, bold claim),
   Setup (2-4s), Payoff/Story (4-10s), Core Value (10-20s), Repeatable
   Device ("3 reasons", "one mistake"), Closing Beat (punchline, challenge,
   callback, or CTA). Beats per 15 seconds is a tracked metric; extreme
   brevity; every line punchy; attention resets constantly.
9. **Lead magnet generator**: extract a liked video's structural patterns,
   tone, teaching frameworks, and themes -> Lead Magnet Blueprint (5-10
   title options, subtitles adding specificity/curiosity/promise, structure:
   Cover, Intro/Promise, Big Idea, Frameworks, Tactical Steps, Mistakes to
   Avoid, Quick Wins, Case Stories, Action Plan/Checklist, CTA) -> full
   2,000-6,000 word draft with checklists, action steps, described diagrams.
   Build frameworks even where the source didn't state them explicitly:
   extract the mental model.

## The conversion-system prompts (#10-#14)

10. **Lead magnet -> call-booking follow-up system**: 7-12 emails + 5-8 SMS.
    Email arc: (1) Delivery + Big Promise, (2) The Gap (knowing vs doing,
    "more info won't fix it"), (3) Case Study/Proof, (4) The Hidden
    Obstacle, (5) Framework Breakdown (expand one framework, show how
    working together accelerates it), (6) Consequences of Doing Nothing
    (light tension + future pacing), (7) Quick Wins Recap + Next Step.
    Optional 8-12: Why Now, FAQ/Objections, soft story reminder, deadline
    nudge, "still interested?", last-chance micro-email. Each email: 3
    subject variations, preview text, soft->medium->strong CTA escalation,
    P.S. with secondary CTA, timing recommendation. SMS: <40 char hook,
    casual, ONE ask, no fluff (delivery text, one-question qualifier, soft
    CTA, credibility bait, direct CTA, check-in/urgency/final nudge).
    Personalization tokens: {first_name} {pain_point} {goal}
    {lead_magnet_title} {booking_link}. Plain-text version of every email.
11. **Offer Architect**: upload any offer -> diagnostic report (strengths,
    weaknesses, missing elements, marketability risks: wrong positioning,
    weak mechanism, low differentiation, value-price misalignment, no risk
    reversal) -> rebuilt Grand Slam version (clearer, faster, lower effort,
    lower risk, more differentiated) with value stack values assigned,
    pricing architecture (anchor high -> optimal -> entry, bundles, payment
    plans), upsell/downsell/cross-sell/order-bump ladder, elevator pitch,
    unique mechanism name, headline options, "here's everything you get"
    bullets. Rule: don't just critique, REBUILD; explain why each change
    raises value or revenue.
12. **Offer -> sales page**: Headline+Subhead (3-5 versions), Pattern
    Interrupt call-out opening, Future Pacing section, "Why You're Stuck"
    (pain, root causes, failed attempts), New Mechanism intro, What Makes
    This Different, Offer Breakdown with [PROOF INSERT] tags, Bonuses,
    bold tangible Guarantee, Pricing (anchor -> real -> justification),
    3 CTA variants, FAQ covering objections.
13. **Offer -> sales call script**: Open/Frame -> Discovery (pain, goals,
    root cause, time horizon, cost of inaction, emotional resonance) ->
    Transition (confirm want, confirm fit, build anticipation) -> Pitch
    (promise, mechanism, how it works, differentiators, deliverables,
    bonuses, guarantees; simple and conversational) -> Proof placeholder ->
    Pricing (anchor -> real -> rationale) -> direct Close ("do you want
    help with this?") -> per-objection mini-scripts (label, validate,
    reframe, present solution) -> Final Close with a clean yes/no.
14. **Offer -> VSL**: Pattern Interrupt Hook -> Big Promise -> Identify the
    Enemy -> Your Story/Credibility -> Big Discovery (mechanism) ->
    Teaching (new way, why old way fails, 2-3 insights that make you the
    inevitable choice) -> Soft Proof [INSERT PROOF] -> Offer Reveal ->
    Value Stack -> Bonuses -> Guarantee -> Price Reveal (anchor -> real ->
    justification) -> Strong CTA -> Future Pacing -> Tension Close (a clear
    decision moment). Must sound spoken, tell a story, avoid hype.

## The doctrine this encodes (why it matters to us)

- **The offer is upstream of everything.** Prompts #12-#14 all consume the
  Offer Architect output. Our pipeline mirror: foundation docs (offer) are
  generated FIRST and every downstream deliverable consumes them.
- **Real artifacts beat imagination.** Winning transcripts -> mother script;
  lost calls -> objection guide; liked videos -> content templates. When a
  client has transcripts, videos, or past winners, MINE them before writing.
- **Proof is placeholdered, never fabricated.** [PROOF INSERT: type + what
  it proves] is the standard; matches our [PROOF:] rule exactly.
- **Anchor -> real price -> justification** is the universal price-reveal
  pattern across page, call, and VSL.
- **Label -> validate -> reframe -> present solution** is the universal
  objection-handling micro-structure (pairs with hormozi-closing.md).

## Where this maps in our pipeline

- **Foundation docs**: offer doc gets the Offer Architect diagnostic lens
  (weaknesses, missing elements, mechanism name, pricing architecture).
- **VSL deliverable**: cross-check our VSL against the 15-beat #14 flow and
  the #5 transcript-derived blueprint; every beat present or consciously cut.
- **Funnel copy**: sales page section order from #12 when long-form pages
  are needed; FAQ = written objection mini-scripts.
- **Email sequences**: the #10 arc is a second reference arc next to
  emails-and-booking.md; SMS rules govern any text/DM scripts we write.
- **Sales scripts**: #13 flow merged with suby-doctor-close.md structure and
  hormozi-closing.md objection arsenal.
- **Onboarding**: when clients provide call recordings or videos, run the
  mining prompts' extraction categories to build voice profiles and
  objection guides before generating any copy.
