# Learnings: breakout-video-writer

Accumulated craft lessons applied on every run.

<!-- 2026-07-09 · job 60002 · client: Trent Kus -->
- The FAQ four-beat framework makes a real client proof point mandatory on every Pillar 1 video, but regulated or early-stage clients often have a small, fixed, screenshot-gated proof set (here just three or four named wins). Forcing the same handful onto 15-plus videos reads as repetitive and thins each win's impact. Add guidance: distribute the named wins across the library so each anchors a different video, and where no fresh proof point fits, allow a mechanism/first-principles backing to satisfy the beat rather than reusing the same result again.

<!-- 2026-07-09 · job 90003 · client: Trent Kus -->
- The skill mandates bullets over scripts because founders read scripts robotically, which is right by default. But some deliverable specs (and some agencies) require word-for-word scripts. Add guidance that the FAQ four-beat (Question → Answer → Why-now → Social proof) is the same scaffold either way: when a word-for-word script is required, ship it with an explicit "freestyle from these beats, do not read verbatim" recording note so the anti-robotic intent survives the format change.
