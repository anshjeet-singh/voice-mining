# Learnings: funnel-page-builder

Accumulated craft lessons applied on every run.

<!-- 2026-07-09 · job 60002 · client: Trent Kus -->
- There is no mode for the VSL landing page itself, yet a call funnel (Branch B of the orchestrator) always needs one: the page the VSL sits on with headline, video embed, single booking CTA, proof strip, and mechanism block. LEAD_MAGNET, POST_CALL_PAGE, PRE_CALL_HUB, and CALENDLY all assume the lead is already past the VSL. Add a VSL_PAGE mode (or an explicit note) so the writer has a defined structure for the top-of-funnel VSL page instead of improvising it.
- POST_CALL_PAGE and PRE_CALL_HUB both live after the booking and overlap heavily, but the skill never states how they relate in sequence. Clarify that the post-call page is the immediate confirmation page (headline confirming the booking, call-confirmed video, offer-breakdown video, a few breakouts, on-page wins) and the pre-call hub is the deeper linked page reached from it (wall of wins, profile-sorted interviews, full offer walkthrough), so a writer producing both knows the intensity ramp and does not duplicate sections.

- There is still no VSL_PAGE mode, yet a Branch B call funnel always needs one and the funnel-structure framework already specifies its 14-section anatomy. Add an explicit VSL_PAGE mode that points the writer to the framework's spec, so the top-of-funnel VSL landing page is built to structure instead of improvised. LEAD_MAGNET, POST_CALL_PAGE, PRE_CALL_HUB, and CALENDLY all assume the lead is already past the VSL.
- The CALENDLY mode defines the four booking questions but the funnel-structure framework's Page 2 (booking page) anatomy does not reference them, so a writer building the booking page from the framework alone can miss them. Cross-reference the two: the booking page's four questions ARE the CALENDLY set, and they belong on Page 2, not on a separate step.
