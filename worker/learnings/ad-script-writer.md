# Learnings: ad-script-writer

Accumulated craft lessons applied on every run.

<!-- 2026-07-10 · job 180009 · client: Trent Kus -->
- The skill's "Output format" only specifies per-ad scripts, but the pipeline also asks for a standalone "Ad Angle Matrix" document (the diversification-engine output as a deliverable in its own right, before any full script is written). Add an output template for that case: (1) the 2 to 4 sub-avatars as a table, (2) the 30+ buying reasons across the ten categories as a table tagged to sub-avatar, (3) the ranked selected angles as a table, each tagged with sub-avatar, buying-reason category, hook category, production format, and awareness level, with the market verbatims and a written hook attached, and (4) the batch verification checklist rendered as a coverage table. Right now an operator has to infer the document shape from the reference file.
- The diversification-engine reference lists production formats (direct-to-camera, whiteboard, podcast, green-screen, demonstration, b-roll, UGC, comment-pin, walking) but the richer native-static taxonomy (the 42 native formats, "would a mate screenshot this" test, platform-to-avatar matching) lives only in static-and-broll-ad-writer/references/native-formats.md. A one-line cross-reference from the diversification engine to that file would help the angle-writer pick a genuinely native format per angle instead of defaulting to talking head.

<!-- 2026-07-10 · job 180009 · client: Trent Kus -->
- The skill and its diversification-engine reference own a verification checklist ("3+ sub-avatars, 5+ hook categories, 5+ production formats, no hook flows onto another ad's body") but there is no worked example of a checklist-passing batch, so a fresh run has to re-derive the pass criteria each time. Add a one-line mapping table example (Ad N: sub-avatar / emotional category / format archetype / production format) showing a 10-ad batch that satisfies all boxes, so the writer can self-check by filling the table rather than re-reasoning the whole checklist.
- The Output format section says "No shot directions unless asked (this is the spoken ad; the editor handles visuals)," which conflicts with regulated/short-form clients whose brand doc mandates text-on-screen and a proof-screenshot gate. Add a carve-out: when the brand requires on-screen proof or the client is a poor teleprompter reader, ship a one-line per-ad recording note (production format + freestyle-from-beats + which `[PROOF: ...]` overlay fills the frame), because that is script-native, not a shot list.

<!-- 2026-07-10 · job 210001 · client: Trent Kus -->
- The skill has no regulated-niche compliance gate in its own body. For finance, credit, and health clients the ad copy must strip personal-attribute claims ("your bad credit"), guaranteed amounts/timelines, quantified approval-adjacent figures, and provider-performance ratios ("nine times out of ten"), replacing them with capability statements and "run your own numbers" invitations. Add a short "Regulated-niche compliance pass" step so a writer who reads only this skill (not the orchestrator stage instructions or the meta-ad-restrictions learnings) doesn't ship non-compliant hooks.
- Output format says "no shot directions unless asked," but for founder-choke clients who freestyle (and for any brand that mandates on-screen proof) a one-line per-ad recording note (production format + freestyle-from-beats + which [PROOF] overlay fills the frame) is load-bearing, not a shot direction. Worth naming as an allowed carve-out in the skill so it isn't stripped by the default rule.

<!-- 2026-07-11 · job 240001 · client: Trent Kus -->
- In regulated niches especially, every ad must state clearly what it is about and who it is for, so both Meta's classifier and its targeting can place it. Reject hooks and headlines whose self-description could sit on any market, avatar, or offer ("show me a check," a bare number): they mis-target and draw manual review. The identification hook must name the specific situation, product category, or mechanism, not a universal feeling.
