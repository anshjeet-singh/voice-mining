# Learnings: course-builder

Accumulated craft lessons applied on every run.

<!-- 2026-07-09 · job 30001 · client: Trent Kus -->
- Template 4's depth standard ("every lesson written to the worked-example level") applied to a full multi-stage curriculum produces an unwieldy document. Add guidance to write one flagship lesson per program to full worked-example depth (the template already names "Read Your Report Like A Lender" as the exemplar) and the remaining lessons to a compact but complete structured block covering the same required fields. This keeps the deliverable usable without dropping any required field.

<!-- 2026-07-13 · job 330001 · client: Blake Matthews -->
- The MENU output format (COURSE / Promise / TIER / Module title + description / Throughline) does not match the agency's fixed Course Outline template that the mother skill actually requires: Recording Standards, a Part 1 free 5-video sales course, a Part 2 paid curriculum in Stage/Module/Lesson blocks where every lesson carries what-to-say / what-not-to-say / tool / win, and a Recording Order. A writer following only course-builder MENU produces a generic tiered menu and silently misses the required shape. Encode the fixed template (or a pointer to it) directly in the MENU output spec so the two do not diverge.
- The "what not to say" field is not just the common mistake. For regulated niches (health, finance, peptides, supplements) it must default to carrying the compliance red line for that lesson, since the outline is what the coach films from and an off-hand claim on camera is a real liability. Add a note that in regulated niches the "what not to say" field leads with the compliance guardrail.

<!-- 2026-07-13 · job 330002 · client: Blake Matthews -->
- When the client's core offer is a done-for-you build (a business, a funnel, an asset handed over) rather than a course, the paid "curriculum" is the operator playbook that ships inside the build. Do not write "how to build the thing" lessons, because the thing is already built. Structure the whole curriculum around the few jobs the operator actually owns after handoff, and open with a "you bought a build, not a course" reframe so the student stops expecting build instructions. This is true for any DFY-build client, not just this one.

<!-- 2026-07-14 · job 390001 · client: Blake Matthews -->
- The fixed Course Outline template requires a "Part 1 free 5-video sales course," but neither the MENU output spec nor the fixed-template note says how to STRUCTURE those five videos, so each writer improvises the arc. Encode the default shape: the five free videos map to the buyer's belief sequence (Video 1 opportunity/dream outcome, Video 2 the real problem mechanism, Video 3 the solution mechanism / Big Domino, Video 4 the vehicle plus proof / DFY-vs-DIY, Video 5 the reframe plus CTA to the workshop or call), and each free video carries goal / what-to-say / what-not-to-say / CTA. This mirrors the RMBC lead-story-problem-solution-close arc and keeps the free course a coherent sales asset, not five loose topics.
- The Course Outline needs a "Recording Order" section, and the useful default is not simply stage order. Film the Part 1 free sales course FIRST (it is the funnel's sales engine and nothing sells until it exists), then the Start Here reframe, then FRONT-LOAD the load-bearing growth stage (the stage that actually drives the client's result), then the remaining stages, ending on the stage the offer's transformation ends on. Add this priority logic to the Recording Order guidance so writers do not default to a flat 1-2-3 stage sequence.
