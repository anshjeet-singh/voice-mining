# Learnings: course-builder

Accumulated craft lessons applied on every run.

<!-- 2026-07-09 · job 30001 · client: Trent Kus -->
- Template 4's depth standard ("every lesson written to the worked-example level") applied to a full multi-stage curriculum produces an unwieldy document. Add guidance to write one flagship lesson per program to full worked-example depth (the template already names "Read Your Report Like A Lender" as the exemplar) and the remaining lessons to a compact but complete structured block covering the same required fields. This keeps the deliverable usable without dropping any required field.

<!-- 2026-07-13 · job 330001 · client: Blake Matthews -->
- The MENU output format (COURSE / Promise / TIER / Module title + description / Throughline) does not match the agency's fixed Course Outline template that the mother skill actually requires: Recording Standards, a Part 1 free 5-video sales course, a Part 2 paid curriculum in Stage/Module/Lesson blocks where every lesson carries what-to-say / what-not-to-say / tool / win, and a Recording Order. A writer following only course-builder MENU produces a generic tiered menu and silently misses the required shape. Encode the fixed template (or a pointer to it) directly in the MENU output spec so the two do not diverge.
- The "what not to say" field is not just the common mistake. For regulated niches (health, finance, peptides, supplements) it must default to carrying the compliance red line for that lesson, since the outline is what the coach films from and an off-hand claim on camera is a real liability. Add a note that in regulated niches the "what not to say" field leads with the compliance guardrail.
