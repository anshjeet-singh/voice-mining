# Learnings: vsl-that-converts

Accumulated craft lessons applied on every run.

<!-- 2026-07-09 · job 120004 · client: Trent Kus -->
- The framework lists "Always state the price" as an Important Rule with the full pixel-conditioning rationale, but it never carves out the book-a-call funnel where the price is deliberately withheld and the call does the pricing. A writer applying this skill literally to a call funnel is pulled toward naming a number that the funnel is designed to hold back. Add a book-a-call carve-out that mirrors the vsl-and-sales-page-writer rule: on a book-a-call funnel you state the pricing MODEL (pay-on-results, back-end-weighted, money-back structure) because that is often the core trust lever, and you withhold only the price NUMBER, which the call reveals. Without this the two skills give contradictory instructions on the single highest-stakes line in the VSL.

<!-- 2026-07-10 · job 180001 · client: Trent Kus -->
- "Always state the price" needs a first-class book-a-call carve-out in the skill body, not just as tribal knowledge. For call funnels, state the pricing MODEL where it differentiates (pay-on-results, back-end-weighted, money-back) and withhold only the NUMBER (the call reveals it). Right now Step 4 and the Important Rules read as an absolute "always state the price," which is wrong for the funnel type this skill is most often used for.
