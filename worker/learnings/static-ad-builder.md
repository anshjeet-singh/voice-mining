Accumulated craft lessons applied on every run.

<!-- 2026-07-10 · job 180009 · client: Trent Kus -->
- The skill documents only the SVG + rsvg-convert + components.py pipeline, but the live agency build route is the ad-factory HTML pipeline: one HTML file of `.s` design elements at 360x450, rendered to 1080x1350 via puppeteer (`render.js`, deviceScaleFactor 3), plus `genimage.js` (Nano Banana Pro) for photo-real formats. Worth documenting as the supported alternative so a builder does not have to reverse-engineer it.
- Add a hard warning about CSS class collisions when many `.s` ads live in one file: an unscoped utility class reused across ads (e.g. a generic `.big` used both as a full-ad wrapper and as an inline number style) bleeds background, padding, and flex onto the wrong element and silently overflows the crop. Rule: scope every per-ad class under that ad's id or a unique prefix, and pixel-audit each render for right-edge clipping.
- For lifestyle / photo + caption-chip formats, recommend generating the photo with genimage (text-free) and compositing the caption chips as HTML over a `file://` background, then rendering. Baking caption text into the image model is unreliable; the HTML overlay guarantees text fidelity, which the format needs.

<!-- 2026-07-10 · operator rejection · client: Trent Kus · batch AdsBatch_2026-07-10 -->
- Operator graded the batch 5/10 and rejected it. The four failures, now codified as hard gates in frameworks/static-render-rules.md: (1) hand-rolled HTML chrome instead of this skill's own pipeline (scripts/ + assets/chrome real-device crops + fonts): for covered formats the skill's pipeline is mandatory, ad-factory is fallback only; (2) designed colored CTA text ("link below" in brand green/blue) baked onto native formats: CTAs live ONLY in the format's designated CTA slot (marker line, final chat bubble, last caption chip) styled per spec; (3) an AI-generated person stood in for the founder on the lifestyle ad: never generate humans, real client photos or a no-person composition only; (4) a tweet ad shipped with a dead white void because no one looked at the render: the visual QA loop (render, VIEW the PNG, grade against reference screenshots and the native test, fix, re-render) is mandatory per ad.

<!-- 2026-07-10 · job 210001 · client: Trent Kus -->
- The QA step (Step 4) checks horizontal extents (`qa_extents`, nothing past x=1012) but has no check for VERTICAL overflow. Fixed-height phone-native formats (iMessage, WhatsApp, IG DM) with `overflow:hidden` silently clip the last message or the footer with no error, dropping the closing beat. Add an explicit clip check to the QA loop: render, then confirm the last authored element is fully visible, or use flow layout that grows the canvas instead of a hard-coded height.
- The genimage guidance for lifestyle/story formats should state plainly: environment-only, no person, no figures, no silhouettes. The image model inserts a human by default even when the brief says "founder from behind," which collides with the "NEVER AI-generate a human" red line. Reserve any human founder shot for the client's real supplied photo, and spec the AI-generated frame as objects and environment only.
