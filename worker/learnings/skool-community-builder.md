- The skill's own name and header brand it "Full Stack Agency," but the mother skill (client-onboarding-orchestrator) mandates that the agency is "Cashflow Coaches" and that "Full Stack Agency" must never appear in any deliverable. Update the skill's branding to Cashflow Coaches (or add a note that the internal name is legacy and never ships) so a writer following this skill doesn't leak the wrong agency name.
- FULL BUILD produces one community per run, but many client funnels need a matched pair: a free top-of-funnel community and a paid delivery community for the offer. The skill treats free-vs-paid only as a VSL variable. Add an explicit "dual build" note covering how to name the pair so ascension reads cleanly, how to differentiate their About pages (free sells one action, paid sells the offer/price/guarantee), and how to link the free community's ascension line to the paid group.
- The membership-question segmenter options are written for a pre-funding credit lead. For a PAID membership the segmenter should split by in-program stage (e.g. repair stage, close-to-fundable) rather than journey-to-entry. Add one line telling the writer to re-cut the segmenter options to the community's tier and the member's actual state on entry.
- Section 4 hands the VSL to `vsl-and-sales-page-writer`, but that skill is not always in scope for the Skool stage (it is a Step 1/Step 3 child in the orchestrator). State that when the handoff target is unavailable, this skill writes the VSL directly from the archetype brief here and still runs the humanizer/dimensionalization chain, so the step never stalls.
